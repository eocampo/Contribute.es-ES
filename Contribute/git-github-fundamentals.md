- <span data-ttu-id="eb8cd-120">*Rama*: unidad de almacenamiento que contiene los archivos y las carpetas que componen el conjunto de contenido de un proyecto.</span><span class="sxs-lookup"><span data-stu-id="eb8cd-120">*Branch*: A unit of storage that contains the files and  folders that make up a project's content set.</span></span> <span data-ttu-id="eb8cd-121">Las ramas separan los flujos de trabajo, a los que se suele hacer referencia como versiones.</span><span class="sxs-lookup"><span data-stu-id="eb8cd-121">Branches separate streams of work (typically referred to as versions).</span></span> <span data-ttu-id="eb8cd-122">Las colaboraciones siempre se realizan en una rama específica, a la que están limitadas exclusivamente.</span><span class="sxs-lookup"><span data-stu-id="eb8cd-122">Contributions are always made and scoped to a specific branch.</span></span> <span data-ttu-id="eb8cd-123">Todos los repositorios contienen una rama predeterminada (que suele denominarse "principal") y una o varias ramas destinadas a fusionarse en la rama "principal".</span><span class="sxs-lookup"><span data-stu-id="eb8cd-123">All repositories contain a default branch (typically named "master") and one or more branches that are destined to be merged back into the master branch.</span></span> <span data-ttu-id="eb8cd-124">La rama principal sirve como la versión actual y la "única fuente de confianza" del proyecto.</span><span class="sxs-lookup"><span data-stu-id="eb8cd-124">The master branch serves as the current version and "single source of truth" for the project.</span></span> <span data-ttu-id="eb8cd-125">Se trata del elemento principal a partir del que se crean todas las demás ramas del repositorio.</span><span class="sxs-lookup"><span data-stu-id="eb8cd-125">It's the parent from which all other branches in the repository are created.</span></span>

<span data-ttu-id="eb8cd-126">Los colaboradores interactúan con Git para actualizar y manipular los repositorios tanto a nivel local como en GitHub:</span><span class="sxs-lookup"><span data-stu-id="eb8cd-126">Contributors interact with Git to update and manipulate repositories at both the local and GitHub levels:</span></span>

- <span data-ttu-id="eb8cd-127">La colaboración puede realizarse de forma local con herramientas como la consola de Git Bash, que admite los comandos de Git para administrar los repositorios locales y establecer conexiones con los repositorios de GitHub.</span><span class="sxs-lookup"><span data-stu-id="eb8cd-127">Locally through tools such as the Git Bash console, which supports Git commands for managing local repositories and communicating with GitHub repositories</span></span>
